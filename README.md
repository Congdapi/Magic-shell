# 🐚 마법의 소라고둥

사주 · 자미두수 · 점성술 데이터를 기반으로 사용자의 명반을 분석해, AI가 맞춤 감정 위로와 조언을 전하는
**AI 심리 코칭 서비스**의 소개 및 고객 유입용 랜딩 페이지입니다.

---

## 1. 기술 스택 (Tech Stack)

| 영역 | 사용 기술 | 비고 |
|---|---|---|
| 마크업 | HTML5 | 시맨틱 태그(`header`, `main`, `section`, `nav`, `footer`) 및 웹 접근성(ARIA) 준수 |
| 스타일 | CSS3 (Vanilla) | `:root` CSS Custom Properties 기반 디자인 토큰 시스템, Flexbox / Grid 레이아웃, `@media` 반응형 |
| 스크립트 | Vanilla JavaScript (ES6+) | 프레임워크 · 빌드 도구 없이 순수 JS로 캐러셀·모달 구현 |
| 폰트 | Pretendard (CDN, `jsdelivr`) | 디자인 가이드 지정 시스템 폰트 |
| 이미지 | Unsplash(히어로 배경) + 자체 에셋(`assets/conch-shell.png`) | 별도 이미지 최적화 파이프라인 없음 |
| 배포 | GitHub Pages | 빌드 과정 없이 정적 파일 그대로 서빙 |

빌드 도구(Webpack, Vite 등)나 프레임워크(React, Vue 등)를 사용하지 않은 **순수 정적 사이트**로, 별도 설치 과정 없이
브라우저에서 `index.html`을 열거나 GitHub Pages에 그대로 업로드해 구동할 수 있습니다.

## 2. 폴더 구조

```
MAGICSHELL/
├── index.html          # 페이지 전체 마크업 (엔트리 포인트)
├── css/
│   └── style.css       # 디자인 토큰 + 레이아웃 + 반응형 + 인터랙션 스타일
├── js/
│   └── script.js       # 캐러셀, 라이트박스 모달 등 인터랙션 로직
├── assets/
│   └── conch-shell.png # 하단 CTA 섹션에 쓰이는 소라고둥 이미지
└── README.md
```

- `css`, `js`, `assets`를 역할별 폴더로 분리해 유지보수 시 원하는 파일을 빠르게 찾을 수 있도록 구성했습니다.
- 이미지를 더 추가할 경우 `assets/` 폴더 안에 넣고 `index.html`에서 상대 경로(`assets/파일명`)로 참조하면 됩니다.

## 3. 정보 구조도 (Information Architecture)

한 페이지 안에서 스크롤로 이어지는 싱글 페이지 랜딩 구조이며, 헤더 내비게이션과 앵커 링크(`#id`)로 각 섹션에 바로 이동할 수 있습니다.

```
마법의 소라고둥 (index.html)
│
├─ Header  (sticky)
│   ├─ 로고
│   └─ CTA 버튼 → #hero-cta
│
├─ ① Hero            [id: top]
│   ├─ 배경 캐러셀 (밤하늘 이미지 3장, 자동재생 + 수동전환)
│   ├─ 서비스 배지 · 헤드라인 · 서브카피
│   ├─ CTA: 첫 상담 무료로 시작하기 (#service)
│   ├─ 링크: 이용 방법 알아보기 (#process)
│   └─ 신뢰 지표: 만족도 / 대기시간 / 무료 여부
│
├─ ② 서비스 안내      [id: service]
│   └─ 4가지 방식 카드: 명반 분석 · AI 감정상담 · 자기이해 인사이트 · 커뮤니티
│
├─ ③ 이용 과정        [id: process]
│   └─ 4단계 스텝: 계정생성 → 명반차트 생성 → 고민입력 → 맞춤 조언
│
├─ ④ 사용자 후기      [id: reviews]
│   └─ 후기 카드 3종 (클릭 시 라이트박스 모달)
│
├─ ⑤ 최종 CTA
│   └─ 소라고둥 이미지 + "마법의 소라고둥과 대화하기" CTA
│
└─ Footer
    ├─ 로고 · 태그라인
    ├─ 링크 2단: 서비스소개 / 이용방법 / FAQ · AI상담 / 커뮤니티 / 개인정보보호
    └─ 저작권 표기

+ 라이트박스 모달 (오버레이, 전역에서 동작)
```

## 4. 주요 기능 및 인터랙션

### 히어로 캐러셀
- 밤하늘 이미지 3장이 **5초 간격 자동 재생**되며, 하단 인디케이터(닷)를 클릭하면 즉시 해당 슬라이드로 전환됩니다.
- 마우스를 캐러셀 위에 올리면 자동재생이 잠시 멈추고, 벗어나면 다시 재생됩니다.
- 슬라이드 전환마다 `aria-selected`, `aria-label` 값을 갱신해 스크린리더 사용자도 현재 위치를 인지할 수 있습니다.

### 버튼 / 카드 마이크로 인터랙션
- 주요 CTA 버튼은 호버 시 `scale(1.05) translateY(-2px)`로 통통 튀는 느낌과 함께 그림자가 밝아집니다.
- 서비스 카드, 후기 카드, 소라고둥 이미지는 호버 시 위로 떠오르며 살짝 회전(`rotate`)하고 은은한 외곽 발광(glow) 그림자가 추가됩니다.
- 모든 트랜지션은 `cubic-bezier(0.175, 0.885, 0.32, 1.275)` 바운스 이징을 공통으로 사용해 통일감을 줍니다.
- `prefers-reduced-motion`을 감지해 모션 최소화를 선호하는 사용자에게는 애니메이션을 사실상 제거합니다.

### 리뷰 라이트박스 모달
- 후기 카드를 클릭(또는 포커스 후 Enter/Space)하면 해당 후기와 연결된 이미지가 **어두운 반투명 오버레이** 위 중앙에 큰 사이즈로 표시됩니다.
- 모달 우측 상단 X 버튼, 바깥 어두운 영역 클릭, `Esc` 키 세 가지 방법으로 모달을 닫을 수 있습니다.
- 모달이 열리면 포커스가 닫기 버튼으로 이동하고, 닫히면 원래 클릭했던 카드로 포커스가 복귀되는 **키보드 접근성 처리**가 되어 있습니다.
- 모달이 열려 있는 동안 배경 스크롤은 잠깁니다.

### 반응형 레이아웃
| 구간 | 너비 | 레이아웃 |
|---|---|---|
| Mobile | ~480px | 1열, 세로 스크롤 중심 |
| Tablet | 481~1024px | 카드/후기 2열 그리드, 여백 확장 |
| Web/Desktop | 1025px~ | 최대 1200px 중앙 정렬, 카드 4열 · 후기 3열, 타이포 스케일 확대 |

Flexbox와 Grid만으로 배치를 구성해 `position: absolute` 좌표 의존 없이 콘텐츠 양에 따라 자연스럽게 리플로우됩니다.

## 5. GitHub Pages 배포 방법

1. 이 폴더(`index.html`, `css/`, `js/`, `assets/`) 전체를 저장소 루트에 커밋 & 푸시합니다.
2. 저장소 **Settings → Pages** 이동
3. **Build and deployment → Source**: `Deploy from a branch` 선택
4. **Branch**: `main`(배포 브랜치), 폴더: `/ (root)` 선택 후 **Save**
5. 잠시 후 `https://<사용자명>.github.io/<저장소명>/` 에서 확인 가능합니다.

## 6. 커스터마이징 포인트
- **색상 / 타이포 / 여백**: `css/style.css` 최상단 `:root` 변수만 수정하면 전체 페이지에 일괄 반영됩니다.
- **히어로 이미지 교체**: `index.html`의 `.carousel-track` 안 `<img src="...">` 주소를 원하는 이미지 URL로 교체하세요.
- **후기 카드 추가**: `.review-grid` 안에 `<li class="review-card" data-modal-trigger data-img="..." data-name="..." data-role="...">` 블록을 복사해 추가하면 자동으로 모달과 연동됩니다.
